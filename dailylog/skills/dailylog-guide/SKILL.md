# 데일리로그 사용 가이드

## 개요

데일리로그는 연도별 단일 파일로 관리되는 일일 기록 시스템이다. `02_Areas/일지/데일리로그 YYYY.md` 경로에 저장된다.

## 파일 구조

```
# 데일리로그 2026

## 1월

### 1월 2주차 (01/05 - 01/09)

#### 2026-01-08 (목)

##### 회사
- 업무 항목 1
- 업무 항목 2

##### 개인
- 개인 항목

##### 스크랩
- [[링크된 문서]]

##### 아이디어
- 아이디어 메모

---
```

## 계층 구조

| 레벨 | 마크다운 | 용도 | 예시 |
|------|----------|------|------|
| H2 | `##` | 월 | `## 1월` |
| H3 | `###` | 주차 | `### 1월 2주차 (01/05 - 01/09)` |
| H4 | `####` | 날짜 | `#### 2026-01-08 (목)` |
| H5 | `#####` | 섹션 | `##### 회사` |

## 섹션 종류

1. **회사**: 업무 관련 기록
2. **개인**: 개인 생활, 취미, 건강 등
3. **스크랩**: 읽은 글, 저장한 링크 (주로 `[[문서명]]` 형식)
4. **아이디어**: 떠오른 생각, 메모

## 날짜 형식

- **날짜 헤더**: `#### YYYY-MM-DD (요일)`
- **요일**: 월, 화, 수, 목, 금, 토, 일

## /dailylog 커맨드 사용

### 읽기

```
/dailylog read              # 오늘
/dailylog read yesterday    # 어제
/dailylog read 2026-01-15   # 특정 날짜
/dailylog read --week       # 이번 주
/dailylog read --month      # 이번 달
```

### 추가

```
/dailylog add --section 회사 "회의 참석"
/dailylog add --section 개인 --date yesterday "운동"
```

### 요약

```
/dailylog summary --week    # 이번 주 통계
/dailylog summary --month   # 이번 달 통계
```

## 활용 패턴

### 일일 기록

매일 작업한 내용을 해당 섹션에 추가:
```
/dailylog add --section 회사 "프로젝트 A 개발 완료"
```

### 주간 리뷰

이번 주 활동을 요약하고 회고:
```
/dailylog summary --week
```

### 특정 날짜 확인

과거 기록 조회:
```
/dailylog read 2026-01-01
```

## 자동 생성

`/dailylog add` 명령 시 해당 날짜 섹션이 없으면 자동으로 템플릿이 생성된다:

```markdown
#### 2026-01-15 (수)

##### 회사
-

##### 개인
-

##### 스크랩
-

##### 아이디어
-
```

## 링크 패턴

스크랩 섹션에서는 주로 Obsidian 내부 링크 사용:
```markdown
##### 스크랩
- [[GeekNews/기사제목|기사제목]]
- [[읽은 책 제목]]
```

## 주의사항

1. **파일 경로 고정**: `02_Areas/일지/데일리로그 {year}.md`
2. **연도별 분리**: 각 연도마다 별도 파일
3. **날짜 순서**: 최신 날짜가 위쪽 (내림차순)
4. **섹션 순서**: 회사 → 개인 → 스크랩 → 아이디어
